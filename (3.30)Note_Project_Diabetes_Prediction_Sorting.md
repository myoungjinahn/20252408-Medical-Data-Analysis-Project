# **의료 데이터 분석 - 당뇨병 데이터를 활용한 발병 가능성 분류** 참고 프로젝트 제안서

## 1. 프로젝트 개요

### 주제
**임상 수치 데이터를 활용한 당뇨병 발병 여부 분류**

환자의 혈당, 체질량지수(BMI), 나이 등 의료 수치 데이터를 분석하여,  
특정 환자가 **당뇨병에 해당하는지(1) or 해당하지 않는지(0)** 를 구분하는  
이진 분류(Binary Classification) 문제

### 목표

1. **데이터 탐색 및 전처리**
   - 의학적으로 0이 불가능한 수치(혈당, 혈압 등)를 결측치로 판단하고 적절히 처리하며, 각 변수가 당뇨병 발병에 어떤 영향을 미치는지 상관관계를 분석

2. **다중 머신러닝 알고리즘 적용 및 비교**
   - KNN, Naive Bayes, SVM, Decision Tree, Random Forest, Logistic Regression 총 6가지 알고리즘을 적용하여 당뇨병 여부를 예측하는 분류 모델 구축 및 성능 비교

3. **모델 최적화 및 평가**
   - 단순히 모델을 돌리는 것에 그치지 않고, GridSearchCV를 통해 각 모델의 최적 하이퍼파라미터를 탐색하여 예측 성능을 향상

4. **당뇨병 조기 진단의 자동화 가능성 확인**
   - 사람이 일일이 복잡한 임상 수치를 보고 판단하는 대신, 머신러닝 모델을 통해 빠르고 객관적으로 당뇨병 발병 여부를 예측할 수 있는 시스템의 가능성 확인


## 2. 선택 이유

- 혈당, 혈압, BMI 등 임상 수치는 대부분 수치형 벡터로 구성되어 있어, 정형 데이터 기반의 머신러닝 분류 알고리즘과 자연스럽게 맞음
- 당뇨병 데이터는 피처 간 비선형 관계가 많아, 다양한 알고리즘을 비교하며 최적 모델을 탐색하는 실험 설계가 의미 있음
- 6가지 알고리즘을 동시에 비교하는 구조로, 알고리즘별 장단점과 적합성을 직접 확인할 수 있어 학습 프로젝트로 적합
- Kaggle의 공개 데이터셋을 사용해 재현 가능성이 높고 데이터 접근이 쉬움

## 3. 데이터셋 설명

### 원본 데이터

- **출처**: Kaggle — "Pima Indians Diabetes Database" (원천: 미국 국립당뇨병·소화기·신장질환연구소, NIDDK)
- **파일명**: `diabetes.csv`
- **전체 행 수**: 768개
- **컬럼 수**: 9개

### 변수 구성

- **독립 변수(Feature)**: 8개의 수치형 임상 수치 데이터

 - `Pregnancies` 임신 횟수
 - `Glucose` 포도당 농도 (혈당)
 - `BloodPressure` 이완기 혈압
 - `SkinThickness` 삼두근 피부 두께
 - `Insulin` 인슐린 수치
 - `BMI` 체질량지수
 - `DiabetesPedigreeFunction` 당뇨 가족력 지수
 - `Age` 나이

- **종속 변수(Target)**: 1개의 범주형 변수
  - `Outcome` — 당뇨병 여부 (0: 정상 / 1: 당뇨)

### 주요 특이사항

- `Glucose`, `BloodPressure`, `SkinThickness`, `Insulin`, `BMI` 5개 컬럼에 **의학적으로 불가능한 0값 존재** → 실질적 결측치로 처리 필요
- 타겟 변수 `Outcome`의 클래스 비율이 불균형 (정상 > 당뇨) 하여 단순 Accuracy만으로 평가 시 성능 과대평가 위험 존재

### Train / Test 분리

- **분리 방법**: `train_test_split` (scikit-learn)
- **훈련 데이터**: 전체의 **80%** (약 614행)
- **테스트 데이터**: 전체의 **20%** (약 154행) (`test_size=0.2, random_state=0`)
- **훈련 데이터 구조**: 입력 — 5개 임상 수치 / 출력 — Outcome (0 or 1)
- **테스트 데이터 역할**: 모델이 본 적 없는 데이터로 예측값과 실제 정답 비교

## 4. 사용 머신러닝 기법

### 알고리즘: 6가지 분류 모델 비교 실험

 - 1. **KNN** 거리 기반, 주변 k개 이웃의 다수결로 분류
 - 2. **Naive Bayes** 확률 기반, 피처 독립 가정 하에 빠른 분류
 - 3. **SVM** 마진 기반, 두 클래스 간 결정 경계(초평면) 최대화
 - 4. **Decision Tree** 트리 기반, 규칙 기반 분기, 해석이 직관적
 - 5. **Random Forest** 앙상블 기반, 비상관 트리 집합으로 분산 감소 및 정확도 향상
 - 6. **Logistic Regression** 선형 기반, 이진 타겟에 가장 기본적인 분류기

```python
from sklearn.neighbors import KNeighborsClassifier
from sklearn.naive_bayes import GaussianNB
from sklearn.svm import SVC
from sklearn.tree import DecisionTreeClassifier
from sklearn.ensemble import RandomForestClassifier
from sklearn.linear_model import LogisticRegression
```

### 데이터 전처리 기법

1. **결측치 처리 (0값 대체)**
   - `Glucose`, `BloodPressure`, `SkinThickness`, `Insulin`, `BMI` 컬럼에 의학적으로 불가능한 0값 존재
   - 분포 형태에 따라 대체 방법을 다르게 적용
     - 정규분포(`Glucose`, `BloodPressure`) → **평균(mean)** 으로 대체
     - 왜도 분포(`SkinThickness`, `Insulin`, `BMI`) → **중앙값(median)** 으로 대체

2. **피처 선택 (Feature Selection)**
   - Pearson 상관계수 히트맵을 통해 `Outcome`(타겟)과 상관이 낮은 3개 컬럼 제거
   - 제거 컬럼: `BloodPressure`, `Insulin`, `DiabetesPedigreeFunction`
   - 최종 사용 피처: `Pregnancies`, `Glucose`, `SkinThickness`, `BMI`, `Age` (5개)

3. **이상치 처리 및 스케일링**
   - 피처 간 단위 차이가 크므로(`Insulin`: 0~846 vs `Pregnancies`: 0~17) **QuantileTransformer**를 적용하여 분포를 균일화하고 이상치의 영향을 최소화

4. **하이퍼파라미터 튜닝**
   - 최적 파라미터를 찾기 위해 **GridSearchCV** + **RepeatedStratifiedKFold** 교차검증 적용

### 상관관계 분석

모델을 만들기 전, 어떤 변수가 당뇨 여부(`Outcome`)와 가장 밀접한 관련이 있는지 파악하는 기법 적용  
→ `Glucose`, `BMI`, `Age`가 가장 높은 상관관계를 보임

## 5. 성능 평가 분석

### 평가 지표

**Accuracy** 전체 예측 중 맞은 비율
**Precision** 당뇨 예측 중 실제 당뇨인 비율
**Recall** 실제 당뇨 환자 중 맞게 예측한 비율
**F1-Score** Precision과 Recall의 조화평균 (불균형 데이터에서 핵심 지표)
**Confusion Matrix** TP / FP / TN / FN 결과 시각화

> 클래스 불균형 데이터이므로 Accuracy 외 **F1-Score**를 핵심 지표로 함께 사용.  
> 특히 의료 분야에서는 실제 당뇨 환자를 놓치는 것(Recall 저하)이 치명적이므로  
> Precision과 Recall을 모두 반영하는 F1-Score가 가장 적절한 평가 지표임.

### 한계 극복을 위한 추가 시도

- **GridSearchCV 하이퍼파라미터 탐색**: KNN, Naive Bayes, SVM, Decision Tree, Random Forest 5개 모델에 최적 파라미터 탐색 적용

KNN: `n_neighbors`, `p`, `weights`, `metric`
Naive Bayes: `var_smoothing`
SVM: `kernel` (poly/rbf/sigmoid), `C`, `gamma`
Decision Tre: `max_depth`, `min_samples_leaf`, `criterion`
Random Forest: `n_estimators`, `max_features`

- **RepeatedStratifiedKFold 교차검증 적용**: 불균형 클래스를 층화 샘플링하여 각 fold에서 클래스 비율을 유지, 신뢰도 높은 평가 수행 (`n_splits=10, n_repeats=3`)
- **분포 기반 전처리**: 단순 평균 대체가 아닌, 각 컬럼의 분포(정규 vs 왜도)를 파악한 후 평균/중앙값을 선택적으로 적용하여 데이터 왜곡 최소화
